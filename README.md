## The Business Problem

Order conversion was running below expectations at a SaaS FinTech company.
Product and sales stakeholders could see *that* conversion was low, but had
no visibility into *where* users were falling out of the workflow — and no
way to self-serve that answer without filing an ad hoc request to the
analytics team (consuming 5+ hours per week).

**Core question:** Where in the order workflow are users dropping off, and
what product changes will recover the most revenue for the least engineering lift?

## Funnel Analysis

The funnel was built in SQL using CASE WHEN stage flags per order,
then visualized in Power BI and Python.

```
Orders initiated         10,000   (100%)
 └─ Entered workflow      5,100   (51%)    ← 49% never engage
     └─ Login attempted   3,200   (63% of workflow enterers)
         └─ Login success   771   (24.1% of login attempts) ← biggest friction
             └─ Completed  1,840   (18.4% overall)
```

**Root causes identified:**
1. **Pre-workflow drop (49%)** — users start an order but never return. No nudge mechanism exists.
2. **Login friction (75.9% failure)** — incorrect bank credentials; no in-app helper tool.
3. **Post-login abandonment (32%)** — session timeout and unclear confirmation UX.

## Revenue Impact Model

Linear regression on 180 days of daily metrics quantifies the revenue lift
per 1 percentage-point improvement in each lever.

```python
model.fit(X_train, y_train)   # features: [interaction_rate, login_success_rate]
# R² = 0.71
# +1pp login success rate  → +$405/day  → +$147,825/year
# +1pp interaction rate    → +$285/day  → +$104,025/year
```

These numbers prioritize the A/B testing roadmap: go after login success first.

---

## A/B Testing

### Experiment 1 — Bank Login Helper Modal ✅ Significant
| Group | N | Login success | Rate |
|---|---|---|---|
| Control (no modal) | 1,200 | 289 | 24.1% |
| Variant B (modal + routing lookup) | 1,200 | 382 | 31.8% |

- Lift: **+7.7pp** (95% CI: +4.2pp to +11.2pp)
- Chi² = 8.91, p = 0.003 → **ship it**
- Est. annual revenue impact: **+$112,000/year**

### Experiment 2 — Re-engagement Email + SMS ⏳ Running
- Day 6/14 — trending +3.2pp (p = 0.09, not yet significant)
- Observed power: 52% — needs full runtime to conclude

### Experiment 3 — Simplified Bank Connection UI 📋 Planned
- Baseline: 24.1% login success
- Target MDE: 10pp | Required n per arm: **2,147 users**
- Hypothesis: inline validation + auto-detected bank logo reduces abandonment by 10–15pp

## Claude AI Integration

The `aws_lambda/insight_generator.py` function runs every night at 07:00 UTC:

1. **Reads** yesterday's metrics from S3
2. **Computes** deltas vs 7-day rolling average
3. **Calls the Claude API** with the anomaly summary and a structured prompt
4. **Posts** the AI-generated insight to a Slack channel

**Example output:**
> "Login success rate fell 8.3% below its 7-day average yesterday — the largest
> single-day drop this quarter. This coincides with a batch of 340 new Enterprise
> onboarding orders from client ENT-042, which typically have lower login success
> in the first week. Recommended action: trigger the login helper modal by default
> for all new Enterprise accounts, and have the CS team proactively send the bank
> credential guide to the ENT-042 contacts today."

## How to Run This Project

### 1. Clone and install
```bash
git clone https://github.com/YOUR_USERNAME/saas-fintech-funnel.git
cd saas-fintech-funnel
pip install -r requirements.txt
```

### 2. Generate synthetic data
```bash
cd python
python generate_data.py

```

### 3. Run funnel analysis
```bash
python funnel_analysis.py
# → prints conversion rates + prints regression model
# → saves 4 charts to ../docs/figures/
```

### 4. Run A/B test analysis
```bash
python ab_testing.py
# → prints chi-squared results, power, sample size for Exp 3
# → saves ab_test_results.png
```

### 5. Run Databricks ETL (optional — requires Community Edition account)
- Upload the 4 CSVs to DBFS via Databricks UI → Data → Add Data
- Open `notebooks/databricks_etl_pipeline.py` in a new Databricks notebook
- Run all cells → creates `saas_funnel` Delta database with 3 tables

### 6. Set up Power BI dashboard
- Follow `powerbi_guide/POWERBI_SETUP.md`
- Connect to Databricks or import CSVs directly

### 7. Run the Claude AI insight generator locally
```bash
export ANTHROPIC_API_KEY=sk-ant-your-key-here
cd aws_lambda
python insight_generator.py
# → reads data/daily_metrics.csv, calls Claude API, prints insight
```

### 8. Deploy to AWS Lambda (optional)
Configure `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `S3_BUCKET`,
`ANTHROPIC_API_KEY`, and `SLACK_WEBHOOK_URL` as GitHub repository secrets,
then push to `main` — the CI/CD pipeline handles the rest.

## SQL Highlights

Eight production-quality queries in `sql/funnel_queries.sql`:

| # | Query | Technique |
|---|---|---|
| 01 | Overall funnel rates | Aggregation, CASE WHEN flags |
| 02 | Stage-over-stage drop-off | LAG window function |
| 03 | Funnel by client tier | GROUP BY + JOIN |
| 04 | A/B test results | FIRST_VALUE window function |
| 05 | Daily trend + rolling avg | AVG OVER with ROWS frame |
| 06 | Client leaderboard + health segment | RANK, CASE WHEN segmentation |
| 07 | Weekly cohort retention | DATE_TRUNC, DATEDIFF |
| 08 | Revenue opportunity sizing | Arithmetic modeling with NULL-safe division |

## Recommendations

Based on this analysis, I recommend the following product changes in priority order:

**1. Ship the login helper modal (Exp 1 winner)**
Immediately deploy Variant B to 100% of users. At +7.7pp login success,
this recovers an estimated $112K in annual revenue.

**2. Run Experiment 2 to completion**
The re-engagement nudge is trending positive but needs 8 more days.
If it reaches significance, ship email + SMS for inactive orders.

**3. Redesign the bank connection screen (Exp 3)**
The largest remaining opportunity. Requires 4,294 total users across arms.
Prioritize in Q3 with the UX team.

**4. Set up automated client health monitoring**
Clients in the "At Risk" segment (< 10% completion) should trigger a
CS outreach automatically — this alone could recover 5–8% of lost revenue
without any product changes.
